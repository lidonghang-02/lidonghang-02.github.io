# 博客维护约束与回归记录

每次修改本仓库代码前，必须完整阅读本文件；修改完成后必须按照“回归测试清单”验证所有受影响功能。不得只验证当前页面。

## 用户明确要求

1. 文章正文以 Markdown 为唯一源码；`2023/`、`2024/` 源码目录中只能提交 `index.md` 和正文图片，不得手写或提交逐文章 `index.html`。
2. GitHub Pages 使用原生 Jekyll 构建；`reader.html` 是统一文章 layout。Jekyll 在部署阶段生成逐文章 `index.html`，生成文件不得回写文章源码目录或提交到源码分支。
3. 不得随意回退、恢复或覆盖已有代码。尤其禁止为了快速恢复功能而使用 `git restore`、`git checkout`、归档覆盖等方式恢复已删除的文章 HTML。
4. 修改共享 CSS、导航、PJAX、阅读器或目录逻辑时，必须检查首页、文章、文章间跳转、浏览器返回、归档、分类、标签、关于、评论、主题切换和移动端是否受影响。
5. 侧边栏品牌始终显示 `LDH Blog`；不得替换成文章标题。
6. 文章横幅/正文上方显示当前文章标题，不得与 `LDH Blog` 或上一篇标题轮换。
7. 首页不得显示文章目录；从文章跳到首页或其他非文章页面时必须清理目录。
8. 正文页与首页使用同一个主内容水平定位变量；不得分别硬编码导致正文偏左或偏右。
9. 文章目录位于正文背景板右侧，固定在视口右上方，不随正文滚动。
10. 目录链接必须可以跳转到对应标题。
11. 目录宽度根据最长目录副标题自适应，但必须有最大宽度；超过最大宽度时显示滚动条，不得无限撑宽或截断且无滚动方式。
12. 超宽代码块不得撑破正文宽度，应显示横向滚动条。
13. 每篇文章都应有评论区；直接访问与站内无刷新跳转后都必须按当前 URL 初始化评论路径。
14. 正文图片与 `index.md` 同目录存放并使用相对路径引用。
15. 首页文章卡片的标题、时间、分类、标签、来源和简介只能来自该文章 `index.md` 的 Front Matter；字段存在才显示，不得从正文、目录路径或默认值推断。`description` 必须原样显示，不再自动截取正文前 200 字。
16. Markdown 运行时生成的代码块必须保持语法高亮；复制按钮固定在代码容器右上方，不得追加到代码内容末尾。
17. 正文必须由 Jekyll/Kramdown/Rouge 在构建期转换并完成语法高亮；浏览器不得再次使用 Marked 或 Highlight.js 解析、替换或重写已经编译的正文。浏览器端代码增强仅限包装滚动容器、添加语言信息和复制按钮。

## 已发生的问题与原因

### 删除文章 HTML 后正文无法打开

- 原因：静态预览服务器只查找目录下的 `index.html`。
- 正确方案：保留文章目录纯 Markdown，使用根目录共享 `reader.html`，预览服务器在发现 `index.md` 时返回共享阅读器。
- 禁止方案：恢复 29 个文章 HTML。

### 评论区消失

- 原因：纯 Markdown 不能直接执行评论脚本；无共享网页外壳时评论区无法存在。
- 正确方案：评论区属于共享阅读器，不属于逐文章 HTML。

### 文章跳转后评论路径错误或评论区消失

- 原因：PJAX 在更新浏览器 URL 之前初始化新页面，评论仍使用旧路径。
- 正确方案：先 `history.pushState`，再初始化文章；路径变化时清理旧评论实例并按新路径创建。

### 标题与 LDH Blog 轮换

- 原因：曾把侧边栏品牌改成文章标题；模板的 Typed.js、旧页面延时任务和 Markdown 阅读器同时写标题。
- 正确方案：侧边栏品牌固定为 `LDH Blog`；文章标题只写 `#subtitle`、`#seo-header` 和 `document.title`；所有延时任务执行前确认仍是当前文章路径。

### 首页出现文章目录

- 原因：文章目录被移动到 `body` 后，PJAX 只替换 `header/main/footer`，旧目录留在页面中。
- 正确方案：目录节点必须标记 `data-site-detached-toc="1"`；页面替换前以及进入非文章页面时删除该节点。

### 正文页与首页水平位置不一致

- 原因：历史 CSS 中存在多组首页和正文宽度、margin、sidebar 补偿覆盖。
- 正确方案：最终覆盖层只使用共享变量 `--site-main-content-width` 和共享水平框架；正文阅读宽度可以独立，但背景板中心线和首页一致。

### 目录随正文移动或速率不一致

- 原因：旧 sidebar/toc 定位、滚动监听和新 fixed 定位同时生效。
- 正确方案：目录外层唯一使用 `position: fixed`；内部 sidebar 使用 `position: static`；不要在 scroll 事件中重复计算 left/top。

### 目录跳转失效

- 可能原因：目录 hash 链接被 PJAX 当成页面导航拦截，或 Markdown 重渲染后目录仍指向旧标题 ID。
- 正确方案：Markdown 渲染完成后重建目录和标题 ID；同页 hash 链接只执行本地滚动，不触发页面 fetch。

### Windows 中文路径恢复产生乱码文件

- 原因：使用 tar/管道恢复包含中文的 Git 文件。
- 正确方案：禁止用 tar 恢复中文文章路径；不得再恢复逐文章 HTML。

### Markdown 代码块高亮消失、复制按钮移动到右下角

- 原因：旧 Hexo 构建结果已经生成 `figure.highlight` 和 `hljs-*` 标记；Marked 运行时只生成 `pre > code.language-*`。若增强逻辑仍只处理旧结构，就不会触发高亮。把复制按钮直接追加到 `pre` 末尾，再使用浮动定位，会使按钮落在代码末尾。
- 正确方案：运行时为 `pre` 建立 `.code-wrapper`，复制按钮作为容器的直接子元素绝对定位在右上角；高亮逻辑同时覆盖 `figure.highlight code` 和 `pre code.language-*`。

### 构建期 Rouge 高亮被浏览器再次改写

- 原因：Jekyll/Rouge 已在构建阶段生成 `.k`、`.kt`、`.s`、`.c1` 等语法标记，但旧兼容逻辑还会加载 Marked/Highlight.js 或用 JavaScript 重写 `code.innerHTML`，造成正文结构、颜色和复制按钮位置不稳定。
- 正确方案：以 Jekyll 编译结果为唯一正文 HTML；使用文章页专用 `css/rouge.css` 为 Rouge 标记着色；浏览器代码只建立滚动/复制容器，不解析 Markdown、不重新高亮、不重写代码内容。

### 目录文字超出背景、宽度没有按标题变化

- 原因：目录文字测量结果没有计入外层、sidebar 和目录内容区的多层内边距；正文过宽时右侧可用空间也会把目录长期压在最小宽度。
- 正确方案：目录外层与 sidebar 不重复保留水平内边距，横向溢出只发生在 `#toc-body` 内；首页与正文继续共用同一个内容宽度变量，并为右侧目录保留实际可用空间。

### 首页摘要长度不统一

- 原因：首页摘要来自旧静态构建内容，没有统一的运行时长度限制。
- 历史方案：进入首页或 PJAX 返回首页时，对 `.index-excerpt > div` 的可见文本按 Unicode 字符截取前 200 个字符。
- 当前方案：以 Front Matter 的 `description` 为唯一简介来源；没有 `description` 就不生成简介节点，不得从正文生成摘要。

## 回归测试清单

每次修改代码后至少完成：

1. `node --check` 检查所有修改过的 JavaScript。
2. `git diff --check`。
3. 确认源码文章目录 HTML 数量为 0、Markdown 数量为当前文章总数；确认 `_site` 中每篇文章均生成一个 HTML。
4. 对全部构建后文章目录执行 HTTP 检查，均应返回 200、包含共享站点外壳且不依赖运行时请求 `index.md`。
5. 浏览器验证首页：标题为 `LDH Blog`、无目录、无横向溢出。
6. 浏览器验证至少两篇文章：Markdown 正文正确、侧边栏为 `LDH Blog`、横幅为当前标题、评论区存在。
7. 验证文章之间 PJAX 跳转及浏览器返回：标题、正文、目录、评论路径均更新。
8. 验证从文章跳到首页、分类、归档、标签、关于：无残留目录或旧文章标题。
9. 验证目录固定位置及目录链接跳转。
10. 验证长目录项达到最大宽度后出现滚动条。
11. 验证超宽代码块出现横向滚动条且页面本身无横向溢出。
12. 验证日间/夜间模式和桌面/移动端布局。
13. 验证 Markdown 代码存在 `hljs-*` 高亮标记，复制按钮位于代码容器右上方。
14. 验证首页卡片字段与各文章 Front Matter 完全一致：有 `description/date/categories/tags/source` 才显示对应节点，缺失字段不生成节点，且简介不得从正文推断或截断。

如果某项因环境限制无法测试，必须明确说明，不能声称全部通过。
